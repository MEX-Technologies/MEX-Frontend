# Contributing to MEX

Join the MEX team and help build the future of cross-border payments.

## GitHub Repository

MEX is an open-source project. Our codebase is available on GitHub:

**Main Repository**: [https://github.com/MEX-Technologies/MEX-Frontend](https://github.com/MEX-Technologies/MEX-Frontend)

## MEX Challenge Program

The MEX Challenge Program is our way of finding talented developers, designers, and contributors to join our mission of revolutionizing cross-border payments.

### What is the MEX Challenge Program?

The MEX Challenge Program is an initiative where developers can:
- Complete coding challenges
- Earn bounties for successful contributions
- Get recognized for their work
- Have the opportunity to join the MEX team

### Challenge Repository

**Challenge Repository**: [https://github.com/MEX-Technologies/mex-challenge](https://github.com/MEX-Technologies/mex-challenge)

The challenge repository contains:
- Open challenges and tasks
- Bounty information
- Contribution guidelines
- Submission instructions
- Team collaboration opportunities

## How to Get Started

### Step 1: Explore the Challenge Repository

Visit the [MEX Challenge repository](https://github.com/MEX-Technologies/mex-challenge) to see available challenges.

### Step 2: Choose a Challenge

Browse available challenges and select one that matches your skills and interests.

### Step 3: Read the Guidelines

Each challenge includes:
- Detailed requirements
- Acceptance criteria
- Bounty information
- Submission process

### Step 4: Start Contributing

- Fork the repository
- Create a branch for your work
- Complete the challenge
- Submit a pull request

### Step 5: Get Rewarded

- Receive bounties for accepted contributions
- Get recognized in our contributor hall of fame
- Potential opportunity to join the MEX team

## Bounty System

### How Bounties Work

#### Bounty Structure
- Each challenge has an associated bounty
- Bounties are paid upon successful completion
- Payment methods: USDC, USDT, or other agreed methods
- Bounty amounts vary based on challenge complexity

#### Payment Process
1. Complete a challenge
2. Submit your work via pull request
3. Get code review and feedback
4. Make necessary adjustments
5. Receive approval
6. Get paid the bounty

## Collaboration Guidelines

### Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Follow coding standards
- Document your work
- Test thoroughly

### Contribution Process

1. **Discuss First**: Open an issue to discuss major changes
2. **Follow Standards**: Adhere to coding and style guidelines
3. **Write Tests**: Include tests for new features
4. **Update Docs**: Keep documentation current
5. **Submit PR**: Create a clear pull request

### Communication

- **GitHub Issues**: For bug reports and feature requests
- **Discussions**: For questions and general discussion
- **Pull Requests**: For code contributions
- **Email**: For sensitive matters
